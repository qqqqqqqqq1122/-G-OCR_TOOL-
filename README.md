既然你之前已经成功推送过一次，说明你的本地文件夹已经和 GitHub 建立好了连接。日常更新代码的流程非常简单，也就是程序员每天都要敲的**“Git 三板斧”**。

你可以直接按下 `Win + R`，输入 `cmd` 回车打开命令行，然后依次执行以下 4 条命令：

### 🛠️ 极速更新“四步曲”

**第一步：进入你的项目文件夹**
因为你的文件夹在 G 盘，所以要加个 `/d` 参数来切换盘符：

```cmd
cd /d G:\OCR_TOOL

```

**第二步：把所有修改过、新增的文件添加到“暂存区”**
（注意 `add` 后面有一个空格，然后是一个英文句号 `.`，代表当前目录下的所有文件）

```cmd
git add .

```

**第三步：给这次更新写个备注（打包提交）**
引号里的字你可以随便改，比如写成今天加了什么新功能，方便以后回溯：

```cmd
git commit -m "更新：加入了 PaddleOCR 云端大模型双保险版代码"

```

**第四步：一键推送到 GitHub**
直接把打包好的代码发射到云端：

```cmd
git push

```

---

### 💡 常见小插曲（提前预警）：

如果你输入 `git push` 之后报错，提示你不知道推到哪个分支，你可以使用下面这行完整版命令（强制推送到主分支）：

```cmd
git push origin main

```

*(注意：如果你的老版本 Git 默认主分支叫 `master`，就把上面的 `main` 换成 `master`)*

就这么简单！以后你每次改完代码想保存到云端，只需要重复上面 2、3、4 步就可以了。你要不要现在就敲一下试试看，看看最后有没有出现 `100%` 成功的提示？
